## 1. システムの目的

- クリエイター活動の公式ホームページとして信頼感を構築する
- 問い合わせ導線で依頼獲得率を高める
- Store/Fanclub を自然導線として運用する

## 2. 解決課題

- 情報発信（News/Blog/Event）が分散しやすい
- 会員限定情報の公開制御が複雑
- 運用者が CMS と公開導線の関係を把握しづらい

## 3. 想定ユーザー

- 訪問者（閲覧中心）
- 会員（FC限定閲覧・課金管理）
- 運用者（CMS更新・公開管理）

## 4. main / store / fc の役割

- **main**: ブランド訴求と導線のハブ
- **store**: 商品閲覧・購入導線
- **fc**: 会員向け情報と継続課金導線

## 5. 提供機能一覧

- コンテンツ閲覧（一覧 / 詳細）
- 多言語切替（ja/en/ko）
- テーマ切替（light/dark/system）
- 会員制限表示
- Contact / Request 送信
- 決済開始（Store/Fanclub）
- CMS運用（Strapi）

## 6. 非機能要件

- SEO（title/description/canonical/構造化データ）
- 可用性（外部連携失敗時のエラー表示）
- 保守性（modules/lib 集約）
- セキュリティ（CORS, rate-limit, webhook署名検証）

## 7. 多言語・テーマ・運用要件

- 追加文言は `ja/en/ko` 同期
- Light/Dark 表示崩れ防止
- CMS運用で Home のハブ導線を壊さない

## 8. 制約条件・拡張余地・未確定

- 制約: 外部サービス（Clerk, Formspree, Stripe）前提
- 拡張余地: CMSフィールド増加、通知・キャンペーン導線拡張
- 未確定事項: 本番レビュー体制、監視基盤詳細

## 9. 仮定

- バックエンド環境変数テンプレートは欠落しているため、設定値はコードから復元可能という前提で整理した。
