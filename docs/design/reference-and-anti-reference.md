# 参考/非参考ガイド

## 参考にする方向性
- 実装前に design context を言語化する。
- 役割（main/store/fc）ごとに情報密度とCTAを調整する。
- polish（余白/タイポ/階層）を最後の工程として分離する。

## 参考にしない方向性
- 外部サイトの命名・章立て・文章の流用。
- repo の責務差を無視した汎用テンプレ適用。
- 単一の巨大ガイドで全タスクを処理する設計。

## 実務ルール
- 画面変更前に「この画面の役割」と「壊してはいけない導線」を1行で定義。
- 実装後に critique を実施し、テンプレ感/過剰演出/可読性を点検。
